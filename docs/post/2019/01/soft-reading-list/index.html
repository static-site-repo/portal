年假可讀的軟體領域輕讀物

2019-01-17
management
約 1054 字 預計閱讀 3 分鐘 | 阅读

中午時，非 RD 的同事請我推薦適合泛軟體產業從業人員的五份輕讀物，可在春節連續假期時好好補血。

年假，自然不方便啃太過於生硬艱澀、理論宏偉的讀物。我挑選這五份輕讀物：四本書，一組網路系列文章。敘事風格親切，高潮迭起，應該可以帶來不少啟發。

IT 實境小說

《鳳凰專案》，體會一下研發、維運、行銷、安控、財務等部門相互掣肘的慘劇，以及如何以 DevOps「三步工作法」來解決的戲劇性發展。

你還沒有達到解決專案可交付成果、故障處理、審計合規等問題所需要的那種對工作的理解程度。在你對工作的內涵有更好的理解之前，任何關於控制工作的討論都會讓你茫然無措。正所謂：夏蟲不可語冰。

敏捷神話

《Scrum：用一半的時間做兩倍的事》，不要被聳動的書名騙了，這本由 Scrum 發明人之一 Jeff Sutherland 所寫的書，揭露 Scrum 發展過程中的軼事。讀了之後，你會領會更多 Scrum 規則背後的精神。

我在書中提及，Scrum 源自於豐田生產系統，以及空戰的 OODA 循環。你會在接下來的內容中看到，從開發價格實惠、每加侖汽油能跑一百哩的車子，到協助美國聯邦調查局 (FBI) 建立 21 世紀的資料庫系統，任何東西都可以用 Scrum 來打造。

跨出框架，需要練習

《Pepper 開發者從 0 到 1 的創新工作法》，讓不是創業家、自由工作者、研究者的平凡上班族，也能習得創新的具體做法。你可以看到，他是如何有紀律地自我要求刻意練習，才能開發出 Pepper 這種人型機器人。

從 0 到 1 需要的不是「才能」，而是「練習」。

外國月亮

《給力》，軟體／媒體科技業巨頭 Netflix 獨一無二的企業文化，但不要驟下「國情不合」的結論。多讀幾次，試著感受適度的低情境 (low context)、直接否定 (direct negative feedback)、平權式領導 (egalitarian) 風格有什麼殊勝之處。

留住員工、使員工高度投入的方法，應該是招募對此計劃興趣濃厚、且傾向在同一工作待上很長時間的人才，而不是靠著提供他們四種加味水及設置員工睡眠艙來留住他們。

是技術，也是藝術

這原本是【主管的修練】讀書會的輔助讀物，但單獨抽出來閱讀，也行。

系列文 / 專案管理的藝術

  • ⓪ 決策的平衡點
  • ① 拉高視野、見林但不見樹
  • ② 平衡需求、但別想討好所有人
  • ③ 重點掌握、如何抓大放小
  • ④ 主動發現、弭禍於無形之中
  • ⑤ 醜話先說，對事但不對人
  • ⑥ 品質管制、追求標準、一致、與穩定
  • ⑦ 降低流言、合適的訊息分享
  • ⑧ 沒有英雄、人員培育與接班計畫
  • ⑨ 避免發散、一切都跟收斂有關
  • ⑩ 歷史借鏡、能重複應用的才有價值



過完年，再來一場討論會吧。

補充

好友 91 (Joey) 隨後也發表了另一份春節期間的推薦讀物。分成三大類：① 輕讀物，② 重一點，需要系列性連串看完的書，③ 想趁著連假，消化一下大部頭的書。

意者，請去看看那份清單。
